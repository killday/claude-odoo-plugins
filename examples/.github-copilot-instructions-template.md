## Testing Best Practices

### Test Class Patterns
- Use `odoo.tests.TransactionCase` for database tests (automatic rollback)
- Use `odoo.tests.HttpCase` for HTTP endpoint/controller tests
- Use `odoo.tests.SavepointCase` for tests requiring custom transaction control
- Use `odoo.tests.tagged('post_install', '-at_install')` for integration tests

### Test Writing Guidelines
```python
from odoo.tests import TransactionCase, tagged

@tagged('post_install', '-at_install')
class TestSaleOrder(TransactionCase):

    @classmethod
    def setUpClass(cls):
        super().setUpClass()
        cls.partner = cls.env['res.partner'].create({'name': 'Test Partner'})

    def test_order_creation(self):
        """Test that orders are created correctly"""
        order = self.env['sale.order'].create({
            'partner_id': self.partner.id,
        })
        self.assertEqual(order.state, 'draft')
```

### Form Helper Pattern (Safer than create())
```python
from odoo.tests.common import Form

def test_with_form(self):
    """Use Form helper for complex record creation"""
    with Form(self.env['sale.order']) as order_form:
        order_form.partner_id = self.partner
        order_form.order_line.new().product_id = self.product
        order = order_form.save()
```

### Mock External APIs
```python
from unittest.mock import patch

def test_external_api(self):
    """Mock external API calls in tests"""
    with patch.object(type(self.env['sale.order']), '_get_external_data') as mock:
        mock.return_value = {'status': 'success'}
        result = self.order._get_external_data()
```

## Upgrade Migration Patterns

### Odoo 17+ XML View Changes

**Replace `<tree>` with `<list>`:**
```xml
<!-- Old (Odoo 14-16) -->
<tree string="Products">
    <field name="name"/>
</tree>

<!-- New (Odoo 17+) -->
<list string="Products">
    <field name="name"/>
</list>
```

**Replace `attrs` with inline attributes:**
```xml
<!-- Old -->
<field name="amount" attrs="{'invisible': [('state', '=', 'draft')]}"/>

<!-- New (Odoo 17+) -->
<field name="amount" invisible="state == 'draft'"/>
```

### Odoo 18+ OWL 2.0 Changes

**Lifecycle hooks renamed:**
```javascript
// Old (OWL 1.x)
setup() {
    onMounted(() => { /* ... */ });
}

// New (OWL 2.0, Odoo 18+)
setup() {
    this.mounted = () => { /* ... */ };
}
```

**Component import changes:**
```javascript
// Old
import { Component } from '@odoo/owl';

// New (Odoo 18+)
import { Component } from '@web/core/component';
```

## Security Best Practices

### Access Rights
- **Always define `ir.model.access.csv`** for new models
- Include at least: `id,name,model_id:id,group_id:id,perm_read,perm_write,perm_create,perm_unlink`
- Example:
```csv
id,name,model_id:id,group_id:id,perm_read,perm_write,perm_create,perm_unlink
access_sale_order_user,sale.order.user,model_sale_order,base.group_user,1,1,1,0
access_sale_order_manager,sale.order.manager,model_sale_order,sales_team.group_sale_manager,1,1,1,1
```

### Route Authentication
- **Always specify `auth` parameter** in `@http.route()`
- Use `auth='user'` for authenticated endpoints (default is 'public')
- Example:
```python
@http.route('/my/orders', auth='user', type='http', website=True)
def my_orders(self):
    orders = request.env['sale.order'].search([('partner_id', '=', request.env.user.partner_id.id)])
    return request.render('my_module.my_orders_template', {'orders': orders})
```

### Sudo() Usage
- **Avoid `sudo()` without security checks**
- Always validate user permissions before using sudo
- Bad:
```python
def get_order(self, order_id):
    return self.env['sale.order'].sudo().browse(order_id)  # ❌ Security risk
```
- Good:
```python
def get_order(self, order_id):
    order = self.env['sale.order'].browse(order_id)
    if order.partner_id != self.env.user.partner_id:
        raise AccessError("You cannot access this order")
    return order.sudo()  # ✅ Safe after validation
```

### SQL Injection Prevention
- **Never use string formatting for SQL queries**
- Bad:
```python
self.env.cr.execute(f"SELECT * FROM sale_order WHERE id = {order_id}")  # ❌ SQL injection risk
```
- Good:
```python
self.env.cr.execute("SELECT * FROM sale_order WHERE id = %s", (order_id,))  # ✅ Safe
```

## Frontend Development

### publicWidget Pattern (Website)
```javascript
odoo.define('my_module.widget', function (require) {
    'use strict';

    const publicWidget = require('web.public.widget');

    publicWidget.registry.MyWidget = publicWidget.Widget.extend({
        selector: '.my-widget-selector',
        events: {
            'click .my-button': '_onClick',
        },

        start: function () {
            // Initialize widget
            return this._super.apply(this, arguments);
        },

        _onClick: function (ev) {
            ev.preventDefault();
            // Handle click
        },
    });

    return publicWidget.registry.MyWidget;
});
```

### RPC Calls
```javascript
// Odoo 14-16
this._rpc({
    route: '/my/route',
    params: { order_id: orderId },
}).then(function (result) {
    // Handle result
});

// Odoo 17+ (Promise-based)
await this.rpc('/my/route', { order_id: orderId });
```

## Internationalization (i18n)

### Translatable Strings

**Python:**
```python
from odoo import _

# Mark strings for translation
raise UserError(_('Invalid operation'))

# With variables
message = _('Order %s has been confirmed') % order.name
```

**XML:**
```xml
<!-- Translatable text -->
<button string="Confirm Order" type="object" name="action_confirm"/>

<!-- Translatable help text -->
<field name="notes" placeholder="Enter notes here..."/>
```

**JavaScript:**
```javascript
const _t = require('web.core')._t;

alert(_t('Order confirmed successfully'));
```

### RTL (Right-to-Left) Support
- Use `start` and `end` instead of `left` and `right` in CSS
- Use flexbox with `flex-direction: row` for automatic RTL support
- Test with `lang=ar_001` parameter

## Performance Tips

### Database Queries
- Use `search_count()` instead of `len(search())`
- Use `mapped()` for field extraction instead of list comprehension
- Use `filtered()` for recordset filtering
- Batch operations with `create()` for multiple records

```python
# Bad
count = len(self.env['sale.order'].search([('state', '=', 'sale')]))

# Good
count = self.env['sale.order'].search_count([('state', '=', 'sale')])

# Bad
names = [order.name for order in orders]

# Good
names = orders.mapped('name')
```

### Computed Fields
- Use `store=True` for frequently accessed computed fields
- Use `depends()` decorator to specify dependencies
- Use `depends_context()` for context-dependent fields

```python
@api.depends('order_line.price_subtotal')
def _compute_amount_total(self):
    for order in self:
        order.amount_total = sum(order.order_line.mapped('price_subtotal'))
```

## Common Pitfalls to Avoid

1. **Don't use `sudo()` in computed fields** - can cause infinite recursion
2. **Don't forget `@api.depends()` on computed fields** - won't recompute
3. **Don't use `self.env.cr.commit()` in business logic** - breaks transaction safety
4. **Don't hardcode record IDs** - use `ref()` with XML IDs instead
5. **Don't forget to add new files to `__init__.py`** - won't be imported
6. **Don't skip `super()` calls in overridden methods** - breaks inheritance chain
7. **Don't use bare `except:` clauses** - catch specific exceptions
8. **Don't forget to update `__manifest__.py` data files** - changes won't load
